
| Код        | Требования                                                      | Комментарий                                                        |
| ---------- | --------------------------------------------------------------- | ------------------------------------------------------------------ |
| F          | **Функциональные (Functionality)**                              |                                                                    |
| **F**-001  | Клиент может подать заявку на депозит через сайт                | Требуется передача данных в кол-центр и защита персональных данных |
| **F**-002  | Клиент может подать заявку на депозит через интернет-банк       | Необходима интеграция с АБС и СМС-подтверждение                    |
| **F**-003  | Автоматический расчёт ставок по депозитам                       | Требуется перенос логики из Excel в АБС                            |
| **F**-004  | Персонализированные ставки для клиентов интернет-банка          | Интеграция с данными клиента в АБС                                 |
| **F**-005  | СМС-подтверждение операций в интернет-банке                     | Использовать существующий СМС-шлюз                                 |
| **F**-006  | Шифрование передаваемых данных (TLS)                            | Обязательно для защиты персональных данных                         |
| U          | **Удобство использования (Usability)**                          |                                                                    |
| **U**-001  | Интерфейс должен соответствовать корпоративному дизайну         | Для удобства пользователей и поддержания бренда                    |
| **U**-002  | Время отклика интерфейса < 500 мс                               | Оптимизация запросов к АБС и кэширование справочных данных         |
| R          | **Надёжность (Reliability)**                                    |                                                                    |
| **R**-001  | Доступность систем 99,9% (uptime)                               | Требуется отказоустойчивость и резервные ЦОДы                      |
| **R**-002  | Поддержка горизонтального масштабирования интернет-банка        | Для обработки растущего числа клиентов                             |
| **R**-003  | Совместимость с MS SQL и Oracle                                 | Минимизация затрат на новые технологии                             |
| **P**      | **Производительность (Performance)**                            |                                                                    |
| **P**-001  | АБС поддерживает только вертикальное масштабирование            | Ограничение текущей архитектуры                                    |
| **P**-002  | Интернет-банк работает на ASP.NET MVC 4.5                       | Возможны сложности с интеграцией новых технологий (Kafka)          |
| S          | **Поддерживаемость (Supportability)**                           |                                                                    |
| **S**-001  | Разделение доступа к данным между отделами кредитов и депозитов | Требования безопасности банка                                      |
| **S**-002  | Шифрование данных при передаче между системами                  | Защита от утечек                                                   |
| +R         | **+ Ограничения (Restrictions)**                                |                                                                    |
| **+R**-001 | Минимизация изменений в ядре интернет-банка (подрядчик)         | Снижение затрат на доработки                                       |
| **+R**-002 | Использование Kafka для асинхронной обработки заявок            | Перспективное решение, но требует доработки интернет-банка         |
| **+R**-003 | Документирование API для будущего расширения                    | Упрощение дальнейшей интеграции новых сервисов                     |
